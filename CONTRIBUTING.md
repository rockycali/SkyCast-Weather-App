# 🤝 Contributing to ClimaFlow

Thank you for your interest in contributing!

This project is primarily maintained as a personal portfolio project, but contributions, ideas, and feedback are always welcome.

---

## 🛠 Development Workflow

This project follows a feature-branch workflow:

1. Fork the repository (optional)
2. Create a new branch from `main`
   ```
   feature/your-feature-name
   fix/your-fix-name
   ```
3. Make your changes
4. Commit with a clear message
   ```
   feat: add new feature
   fix: resolve issue
   docs: update documentation
   ```
5. Push your branch
6. Open a Pull Request

---

## 📏 Code Style

- Use SwiftUI best practices
- Follow MVVM architecture
- Keep views clean and focused
- Avoid duplicating logic
- Prefer readability over clever code

---

## 💬 Pull Requests

When opening a PR:

- Clearly describe what you changed
- Link related issues if applicable
- Keep PRs focused (one feature or fix per PR)

---

## 📌 Notes

- This project is actively evolving
- UI/UX consistency is important
- Changes may be refined during review

---

## 🙌 Thanks

Even small contributions, ideas, or feedback are appreciated!
